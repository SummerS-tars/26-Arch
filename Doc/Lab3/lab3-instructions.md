  2. **移位指令扩展**：

    - **逻辑右移**：`srl`, `srli`。高位补 0。

### 阶段二：无条件跳转的数据通路与副作用（JAL / JALR）
**目标**：实现无条件跳转，并**绝对不能遗漏**保存返回地址到目标寄存器（Link Register）的副作用。
**易漏点**：JALR 的目标地址最低位清零规则；`rd` 的写回逻辑。

- **具体执行项**：
  1. **J-Type (jal) 处理**：
    - 目标地址计算：`Target_PC = Current_PC + sext(imm)`。
    - 寄存器写回：将 `Current_PC + 4` 写回至指令指定的 `rd`（通常为 `x1` / `ra`）。
  2. **I-Type (jalr) 处理**：
    - 目标地址计算：`Target_PC = (rs1 + sext(imm)) & ~1`（必须强制将最低位设为 0）。
    - 寄存器写回：将 `Current_PC + 4` 写回至指令指定的 `rd`。
- **阶段验收**：`jal` 和 `jalr` 能够正确计算跳转目标地址，且流水线 WB 阶段成功将 `PC + 4` 写入正确的通用寄存器。

### 阶段三：条件分支比较器的全面实现（B-Type）
**目标**：完全覆盖 6 条分支指令的比较逻辑，并生成 `branch_taken` 信号。
**易漏点**：正负数边界条件的判定（如极负数与正数的比较）。

- **具体执行项**：
  1. **提取 B-Type 立即数**：按 RV 规范重组乱序的立即数位（`imm[12|10:5|4:1|11]`），进行符号扩展。
  2. **实现专属分支比较逻辑**（可在 ID 级或 EX 级，建议 ID 级以减少冲刷惩罚）：
    - `beq` (相等)：`rs1 == rs2`
    - `bne` (不等)：`rs1 != rs2`
    - `blt` (有符号小于)：`$signed(rs1) < $signed(rs2)`
    - `bge` (有符号大于等于)：`$signed(rs1) >= $signed(rs2)`
    - `bltu` (无符号小于)：`rs1 < rs2`
    - `bgeu` (无符号大于等于)：`rs1 >= rs2`
  3. **计算分支目标**：`Branch_Target = Current_PC + sext(imm)`。
- **阶段验收**：遇到分支指令时，能够根据当前寄存器状态，100% 准确地拉高或拉低 `branch_taken` 信号。

### 阶段四：控制冒险处理（流水线冲刷 Flush）
**目标**：这是 Lab 3 的**核心挑战**，彻底解决“错误取指”污染系统状态的问题。
**易漏点**：冲刷时未能清除 `MemWrite` 导致内存被误写；只冲刷了 IF 级忘记冲刷 ID 级。

- **具体执行项**：
  1. **确立决断点**：如果分支/跳转判定在 EX 级得出，此时错误指令已进入 IF 和 ID 级（即“预测不跳转”失败）。
  2. **触发 Flush**：当检测到 `branch_taken == 1` 或遇到 `jal/jalr` 时：
    - 向 IF/ID 流水线寄存器发送 `flush` 信号。
    - 向 ID/EX 流水线寄存器发送 `flush` 信号。
  3. **气泡（Bubble）行为定义**：被 `flush` 的流水段寄存器在下一个时钟周期必须输出全 0（相当于 `addi x0, x0, 0` NOP），强行将 `RegWrite` 和 `MemWrite` 置 0。
  4. **PC 强行重定向**：下一个时钟周期，PC 寄存器的输入强制切换为阶段二/三算出的 `Target_PC`。
- **阶段验收**：在波形中确认，紧随发生跳转的 `jal/beq` 后面的两条指令变为 NOP，且系统状态未受这两条错误指令影响。
