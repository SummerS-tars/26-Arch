### 1. 数据前递 / 旁路转发 (Forwarding / Bypassing)
**解决什么问题**：当后一条指令需要使用前一条（或前两条）指令刚计算出、但**还未写回寄存器堆**的数据时（RAW 冒险），不能去读寄存器里旧的值，而是要直接从流水线后段把新鲜出炉的数据“拉”回来用。
**实现指南**：
你需要实现一个**转发控制单元 (Forwarding Unit)**。它通常在 EX（执行）阶段发挥作用，控制 ALU 两个输入端前的多路选择器（Mux）。

- **EX 冒险 (优先级最高)**：
前一条指令的结果在 EX/MEM 寄存器中。
*判断条件*：`EX_MEM.reg_write == 1` 且 `EX_MEM.rd != 0` 且 `EX_MEM.rd == ID_EX.rs1`（对于 rs2 同理）。
*动作*：将 ALU 的输入 A（或 B）切换为 `EX_MEM.alu_result`。
- **MEM 冒险 (优先级次之)**：
前上条指令的结果在 MEM/WB 寄存器中。
*判断条件*：`MEM_WB.reg_write == 1` 且 `MEM_WB.rd != 0` 且 `MEM_WB.rd == ID_EX.rs1`。
*动作*：将 ALU 的输入 A（或 B）切换为 `MEM_WB.wb_data`。
- **⚠️ 致命陷阱：双重数据冒险 (Double Data Hazard)**：
考虑这种情况：
`ADD x1, x1, x2``ADD x1, x1, x3``ADD x1, x1, x4`
最后一条指令的 `rs1` 既匹配前一条（EX 冒险），又匹配前上条（MEM 冒险）。此时**必须优先使用 EX 冒险的数据**（因为它更新）。在写 Verilog 的 `if-else` 逻辑时，务必把 EX 冒险的判断放在前面！

### 2. 流水线阻塞与插入气泡 (Stall & Bubble)
**解决什么问题**：如果前一条指令是 `Load`（如 `lw`），它必须等到 MEM 阶段才能从内存拿到数据。如果紧接着的下一条指令就要用这个数据（Load-Use 冒险），即使有转发网络，时间上也来不及（数据在 MEM 阶段末尾才出来，但下一条指令此时已经在 EX 阶段等米下锅了）。唯一的办法是：让流水线暂停一拍。
**实现指南**：
你需要实现一个**冒险检测单元 (Hazard Detection Unit)**。它通常在 ID（译码）阶段进行判断。

- **检测 Load-Use 冒险的条件**：
当前在 EX 阶段的指令是一条 Load 指令：`ID_EX.mem_read == 1`
并且它的目的寄存器正好是 ID 阶段指令需要的源寄存器：`ID_EX.rd == IF_ID.rs1` 或 `ID_EX.rd == IF_ID.rs2`。
- **如何“暂停” (Stall & Flush)？**
一旦检测到上述条件，必须在下一个时钟上升沿做三件事：
  1. **停住 PC**：禁止 PC 寄存器更新（`PCWrite = 0`），下个周期重新取这条因为没等到数据而无法执行的指令。
  2. **停住 IF/ID 寄存器**：禁止 IF/ID 寄存器更新（`IF_ID_Write = 0`），把卡在译码阶段的指令“按住”。
  3. **清空 ID/EX 寄存器（插入气泡 Bubble）**：既然前两级停住了，不能让错误的控制信号继续往后传。将送往 ID/EX 寄存器的所有控制信号（如 `reg_write`, `mem_read`, `alu_op` 等）强行置为 0（相当于插入了一条什么都不做的 `NOP` 指令）。
